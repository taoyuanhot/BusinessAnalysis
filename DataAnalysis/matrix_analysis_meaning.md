# Matrix Analysis Meaning

## 1. Core Meaning

Matrix analysis is used to examine the relationship between two paper-level categories, rather than only reporting the frequency of each category separately.

In this project, a matrix helps answer:

- Which topics are usually studied with which methods?
- Which academic fields tend to focus on which topics?
- Which topic–method or field–topic combinations are common?
- Which combinations are rare or underdeveloped?

The purpose is not to make the analysis more complicated, but to move from simple frequency counts to structural comparison.

---

## 2. Why Matrix Is Needed

A bar chart can only show how many papers belong to each category.

For example, it can show:

- how many papers discuss customer experience
- how many papers use SEM
- how many papers belong to marketing

However, it cannot show whether customer experience papers mostly use SEM, or whether marketing papers mostly focus on customer experience.

A matrix solves this problem by showing how two categories are connected.

In short:

> Frequency analysis shows “how many.”  
> Matrix analysis shows “which category is connected to which category.”

---

## 3. Topic–Method Matrix

The topic–method matrix shows which research methods are commonly used for each topic.

Example structure:

| Topic / Method | Survey / SEM | Case study | Optimization | Bibliometric |
|---|---:|---:|---:|---:|
| Customer experience |  |  |  |  |
| Logistics / fulfillment |  |  |  |  |
| Inventory / operations |  |  |  |  |
| Digital technology / IT |  |  |  |  |
| Strategy / business model |  |  |  |  |

This matrix answers:

- Are customer experience papers mainly using survey or SEM?
- Are logistics papers mainly using case studies or optimization?
- Are inventory papers mainly using mathematical models?
- Are some topics lacking quantitative methods?
- Are some methods concentrated in only one topic area?

The value of this matrix is to identify methodological patterns and methodological gaps.

---

## 4. Field–Topic Matrix

The field–topic matrix shows which academic fields focus on which topics.

Example structure:

| Main Field / Topic | Customer experience | Logistics | Inventory | IT | Strategy |
|---|---:|---:|---:|---:|---:|
| Marketing |  |  |  |  |  |
| Supply Chain Management |  |  |  |  |  |
| Information Systems |  |  |  |  |  |
| Management Science & OR |  |  |  |  |  |

This matrix answers:

- Does marketing mainly study customer experience?
- Does supply chain management mainly study logistics and fulfillment?
- Does management science mainly study inventory and optimization?
- Does information systems mainly study digital platforms and IT?
- Which topics are shared across multiple fields?
- Which topics are dominated by one field?

The value of this matrix is to identify disciplinary concentration and cross-field opportunities.

---

## 5. Interpretation

Matrix analysis should be interpreted as a structural tool.

It does not say that one topic, method, or field is better than another.

It shows:

- concentration
- imbalance
- dominant combinations
- missing combinations
- potential research gaps
- interdisciplinary connections

For example:

> If customer experience papers are mostly studied through SEM, while few use case studies or experiments, this suggests a methodological concentration.

> If inventory papers are mostly in management science and rarely appear in marketing, this suggests a field-topic separation.

> If digital technology appears across marketing, supply chain management, and information systems, it may function as an interdisciplinary bridge.

For the Gibson papers corpus, use matrix analysis after assigning each paper:

2. Method category
3. Main field
4. Sub-field
5. Cross-disciplinary count

Then create at least two matrices:

1. Topic–method matrix
2. Field–topic matrix

Optional matrices:

- Year–method matrix
- Field–method matrix
- Topic–cross-disciplinary count matrix

---

## 7. Short Method Statement

Matrix analysis is used to examine the relationships between coded paper-level categories. The topic–method matrix identifies how different research topics are methodologically studied, while the field–topic matrix identifies how academic fields distribute their attention across research topics. These matrices reveal dominant combinations, underdeveloped combinations, and possible research gaps within the literature corpus.
